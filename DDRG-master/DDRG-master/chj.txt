* 규칙 정하기 
    - 하나의 branch에서 해서 충돌할 확률이 높음
    - 별도의 소스코드 timeline 만들 수 있음
    - 하나의 branch 복제 , 별도의 timeline
                        -> 나중에 2개 합침

* timeline
    - 나만 이력 남김

* branch 생성하기
    - git branch

* 각자의 branch 만들기
    - git branch branch명
    - git branch chj

    - git branch

* branch 이동
    - git checkout chj
    - (master) -> (chj) 변경됨

* 추가하기
    - git add ./chj.txt
    - git commit -m 'add chj file'


    - git checkout master
    - chj 파일 사라진걸 볼 수 있음
    - 서로 다른 timeline 갖게 함
